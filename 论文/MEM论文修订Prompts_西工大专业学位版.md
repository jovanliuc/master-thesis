# MEM论文修订高质量 Prompts 合集（西工大专业学位版）
> 使用说明：将下面的 prompts 按需复制到对话中，把【占位符】替换为你的内容。所有输出**严格遵循**《西北工业大学研究生学位论文写作指南（专业学位版）》的格式与学术规范。

---

## 总控类（建议每次开工先贴）

(1) 总控-严格按校规排版与学术风格
```
角色：你是具备工程管理+计算机+软件工程背景的大学教授与学术写作教练。
任务：对我之后提供的任何论文内容，**仅按**《西北工业大学研究生学位论文写作指南（专业学位版）》进行写作与排版（A4、边距上/下25.4mm左/右25mm、正文宋体小四/Times New Roman 12pt、固定行距20pt、标题黑体三/四/小四、图表与公式按章编号、摘要第三人称、关键词分号分隔、参考文献GB/T 7714—2015顺序编码制、摘要/目录页码罗马、正文阿拉伯自绪论起等）。
输出：给出“合规版文本+我改动了哪些点”的列表，不要改变事实与数据。若遇到不清楚的格式细节，按校规最严格解读执行。
```

(2) 总控-逐步法（先诊断再修订）
```
第一步：列出该段/该章问题清单（结构、逻辑、证据、格式、语言，用(1)(2)(3)…）。
第二步：给出修订目标与评价指标（如：冗余率<10%、每段有主题句/证据/小结句）。
第三步：产出修订版，并附“对照表”（原句→问题→新句）。
第四步：执行校规合规检查清单（标题级别、行距、图表/公式编号、引用规范等）并逐项确认。
```

---

## 逐章逐节改写

(3) 改写指定小节（保结构可读性）
```
输入：小节标题【如：1.2 国内外研究现状】+ 原文 +（可选）老师批注。
目标：保留事实，重构为“主题句—证据—评述”三层结构；每段首句明确观点，中间支持证据，末句给出评述/启示；控制在【字数】±10%。
产物：修订版 + 逐条对照表 + 后续补证据/补图表的待办列表（含具体建议）。
```

(4) 把老师批注逐条落地
```
输入：原文 + 批注清单（或CSV要点）。
输出：(1) 批注→动作→落地文本 的落地表；(2) 已解决/未解决统计；(3) 未解决项的建议证据来源/数据口径/图表类型。
```

(5) 摘要/Abstract 对齐
```
约束：第三人称；中文关键词以分号分隔；中英术语一致；避免方法与结论重复表述。
输出：中/英文摘要各【字数】；关键词5–8个；给出“不合规示例→修正”2–3例。
```

## 文献与理论

(6) 文献综述瘦身与结构化
```
输入：综述原文。
目标：删除综述型空话，按“主题簇→代表文献→方法→发现→局限→可借鉴点”重写；为每个主题簇给出小结句。
输出：结构化综述 + “述评矩阵”（作者-年份-对象-方法-主要结论-局限-可借鉴点）。
```

(7) 补齐 2023–2025 前沿引用（会联网检索）
```
输入：主题【如：DevSecOps 在银行业落地的安全左移实践】+ 需要的文献数量【如：10】。
输出：最新高质量来源（中英混合），每条含可检索出处、一句话贡献、与你论文的适配性说明；按 GB/T 7714—2015 给出规范条目，并按正文顺序编码制列出引用占位。
```

(8) 参考文献规范化与一一对应
```
输入：文末参考文献列表 + 正文中引用片段（含[1][2]…）。
输出：(1) 修复重复/缺失/顺序错位/不合格格式；(2) 生成合规版参考文献；(3) 生成“引用映射表”（正文位置→[n]→文末条目）。
```

## 方法与数据（工程管理 + 软件工程）

(9) 度量体系设计（Dev/QA/Sec/Ops 四域）
```
目标：构建“输入—过程—输出—结果”分层指标；区分领先/滞后指标；映射到价值流。
输出：指标表（定义、口径、采集频率、仪表板展示、风险阈值、目标值）、示例SQL/脚本或Jira字段映射。
```

(10) 效果评估实验设计
```
输入：目标改进点【如：安全左移门禁+自动化SAST/DAST引入】。
输出：前后对照设计（基线期 vs 试点期）、干扰变量控制、统计检验（如 Mann-Whitney U/χ²/比例差检验）、样本量估计、可视化方案。
```

(11) DSM（设计结构矩阵）建模与解耦建议
```
输入：关键任务清单及依赖说明。
输出：DSM矩阵（任务排序、强连通分量、反馈边识别）、解耦策略、里程碑重排建议、延期率/返工率影响假设。
```

(12) DevSecOps 流程蓝图（门禁 + 责任分离）
```
输入：现状流程泳道图或文字描述。
输出：改进后的泳道图描述（阶段、输入/输出、门禁、自动化任务、RACI）、关键控制点与合规证据链（日志、制品、报告）。
```

## 图表、公式与排版

(13) 三线表/图题批量规范
```
输入：表格/图的题注与正文引用。
输出：(1) 将表格改为三线表并统一“表X-Y/图X-Y”命名；(2) 给出表题居上/图题居下的规范文本；(3) 生成“表图清单”与“交叉引用待修正项”。
```

(14) 公式编号与变量表
```
输入：含公式的小节内容。
输出：按章编号的公式标签【式(3-2)】，变量符号表（符号-含义-单位-取值范围），并检查单位是否符合 GB 3100/3101/3102。
```

## 语言与风格

(15) 学术中文润色（降冗 + 提严谨）
```
目标：删口语化、去主观形容词、减少被动连环句，提升逻辑连接（因此/此外/由此可见），避免解释性循环。
输出：修订版 + 冗余率估算 + 常见问题对照表。
```

(16) 科技英文润色（Abstract/术语一致）
```
目标：术语与中文一致；避免中式被动；确保“主题句-证据-总结”结构。
输出：润色版 + 修改标注（附简短理由）。
```

(17) 压缩到目标字数
```
输入：原文 + 目标字数。
输出：在不损失信息密度前提下压缩；保留所有引用编号；附“被删要点列表”。
```

## 盲审、合规与风险

(18) 盲审脱敏版生成
```
任务：删除/替换个人、导师、企业可识别信息；以“H公司/某银行IT部”等匿名化；保留必要背景。
输出：盲审版 + “脱敏对照表”（原词→替换词）。
```

(19) 学术诚信与相似性风险预检（非查重）
```
任务：发现潜在高重合段落（设定重复阈值与句式雷区），给出重写建议；标出应加引用而未引之处。
输出：风险清单 + 建议重写稿（保证意思不变）。
```

## 答辩与附录

(20) 结论与贡献强化
```
输入：当前“结论”章节。
输出：区分学术贡献与实践贡献；明确可复制性边界与威胁与限制；给出“如果复现实验失败，最可能的3个原因”。
```

(21) 答辩 Q&A 清单（十问十答）
```
输入：论文核心主张+方法+数据来源。
输出：评委常问【10】题（含追问），每题提供1分版/3分版/5分版答案与可视化辅助建议。
```

(22) 附录与在学成果编排
```
任务：将代码/问卷/推导放入附录A/B/C…；“在学成果”列出与论文相关的论文/专利/项目。
输出：合规版附录目录与内容模板。
```

## Word/LaTeX 工程化输出

(23) 导出为合规 Word 并分节分页
```
输入：要导出的章节文本 + 章起始位置（是否自“绪论”起）。
输出：.docx（A4、页边距、固定行距20pt、标题黑体、摘要/目录页码罗马、正文页码阿拉伯从1、页眉“西北工业大学硕士学位论文”、页脚居中页码、图表/公式按章编号）。
要求：输出后附“自动化校规检查清单”并标记已通过项。
```

(24) 参考文献自动重排（GB/T 7714 顺序编码制）
```
输入：正文（含[1][2]…）+ 文末参考文献。
输出：重排后的合规参考文献表 + “引用映射表” + 未匹配项与建议处理。
```

## 示例一键调用

(25) 示例：修 1.2 文献现状
```
请按“总控-逐步法”为我重写【1.2 国内外研究现状】（见下方原文）。
约束：按校写作指南；控制在【1200】±10%字；每段“主题句—证据—评述”；必要处保留[引用编号]；输出“问题清单→修订目标→修订版→对照表→述评矩阵→后续待办（需要新增的证据/数据/图表）”。
原文：
【粘贴原文】
老师意见摘要（可选）：
【粘贴批注要点】
```

(26) 示例：三线表规范化
```
以下是我第【3】章的5个表格文本与题注。请把它们改为三线表样式，统一编号为“表3-1…表3-5”，表题居上；并列出正文中需要修正的交叉引用位置。
表格与题注：
【粘贴】
```

(27) 示例：参考文献一一对应
```
下面是正文片段（含[编号]）与文末参考文献列表。请按 GB/T 7714—2015 顺序编码制重排，并产出“正文位置→[n]→文末条目”的映射表，指出缺失/重复/错位并修复。
正文片段：
【粘贴】
文末参考文献：
【粘贴】
```

## 使用小贴士
(1) 先贴“总控类”确保风格与格式的硬约束始终生效。
(2) 每次限定目标字数与结构，并要求“对照表”，命中率显著提升。
(3) 需要外部文献时，明确写“需要联网检索 2023–2025 高质量来源”。
(4) 表/图/式/参考文献/页码体系必须“按章编号”“顺序编码制”。
(5) 导出 Word 时配合(23)(24)，可直接得到合规 .docx。
